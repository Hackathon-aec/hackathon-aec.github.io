Hackathon Challenge · Problem 2

IFC Model Repair with AI

Given a corrupted IFC building model, detect structural errors and reconstruct a valid, semantically complete IFC file - using a fine-tuned small language model.

🏗️ Background

IFC (Industry Foundation Classes) is the standard file format for building models (BIM). Real-world IFC files may be broken - missing elements, invalid geometry, disconnected structure. Downstream tools crash or produce wrong results when given such files.

Your challenge: build an AI system that takes a corrupted IFC file and repairs it.

📁 The Dataset

You are provided with a small set of sample IFC pairs (clean + corrupted) to help you understand the data format and the types of faults you will encounter. Beyond these samples you are free - and encouraged - to build your own training dataset: download real IFC files from public sources, generate synthetic ones, and inject faults using your own noise injection script.

At evaluation time you will receive 3 corrupted test IFC files. Your model must detect the faults and output a fully repaired IFC file for each.

IFC File Structure

Each .ifc file is a plain text file in STEP format. It has a header and a data section. Below is taken directly from IFC model .ifc in sample_hackathon_dataset_2/: 

ISO-10303-21;
HEADER;
FILE_DESCRIPTION(('no view'),'2;1');
FILE_NAME('Test IFC model.ifc','2026-04-16T20:27:01',('Gaurav Kumar Aggarwal'),('No Organization',''),'ODA SDAI 25.4','','gkaggarwal');
FILE_SCHEMA(('IFC2X3'));
ENDSEC;

DATA;
#1=IFCPERSON($,'Aggarwal','Gaurav Kumar',$,$,$,$,$);
#5=IFCOWNERHISTORY(#3,#4,$,.NOCHANGE.,$,$,$,1776351420);
#6=IFCUNITASSIGNMENT((#7,#8,#9,#10,#14,#20,#28));
#7=IFCSIUNIT(*,.LENGTHUNIT.,$,.METRE.);
#31=IFCPROJECT('1xQcHvQ6j72xuabmuaT4K3',#5,'For IFC testing',$,$,$,$,(#32),#6);
#33=IFCBUILDING('3EDmDtY1P31OpEUrYzkrZb',#5,'Building',$,$,#36,$,$,.ELEMENT.,$,$,$);
#37=IFCBUILDINGSTOREY('1I$OEmFwb0dfLQlFygoNz8',#5,'Ground floor',$,$,#40,$,$,.ELEMENT.,0.);
#482=IFCRELCONTAINEDINSPATIALSTRUCTURE('3pao28ZO50aA8IifzHISnu',#5,$,$,(#67,#94,#121,...),#37);
#509=IFCCOLUMN('1nChuMS3z3deKfNve4ghPR',#5,' ',$,$,#485,#486,$);
...
ENDSEC;
END-ISO-10303-21;

Each line starts with an entity ID (#N), followed by the entity type and its attributes. References like #37 point to other entities by ID. A $ means a null/optional attribute.

IFC has hundreds of entity types — you do not need to handle them all. Focus on the classes present in the sample files (listed above). These cover the core floor-plan elements: spatial containers (IFCPROJECT, IFCBUILDING, IFCBUILDINGSTOREY, IFCSPACE), building elements (IFCWALL, IFCCOLUMN, IFCBEAM, IFCSLAB, IFCDOOR, IFCWINDOW), geometry primitives (IFCEXTRUDEDAREASOLID, IFCPOLYLINE), and the relationship entity (IFCRELCONTAINEDINSPATIALSTRUCTURE) that links elements to their storey. A repair pipeline that handles this subset correctly will cover the test cases.

⚠️ IFC Fault Types

The test IFC files contain one or more of the fault categories below, as observed in the provided sample_hackathon_dataset_2/ files. These categories are illustrative - not exhaustive: real IFC files can exhibit a much wider range of structural and semantic issues. Build a general-purpose repair pipeline.

Study the files in sample_hackathon_dataset_2/ to understand how each fault manifests in raw STEP text.

Observed Fault Categories

Element Clash

Two solid elements (e.g. a beam and a column) occupy overlapping 3-D space. Seen in IFC corner beam intersection.ifc - beams intersect columns at corner nodes in a physically impossible way.

Geometric Inconsistency

An element’s geometry does not match its intended structural role - e.g. a column that is shorter than the storey height it is supposed to span. Seen in IFC corner column shorter.ifc.

Truncated Geometry

IFCPOLYLINE instances with fewer than 3 points, making them invalid as closed profiles. Found across all three sample files - the boundary curves of structural cross-sections are incomplete.

Missing Spatial Elements

A building model contains walls, beams, and columns but no IFCSPACE entities - the spatial decomposition is absent. Seen in IFC model .ifc.

Not limited to the above. The test files may also exhibit missing properties, disconnected floors, dangling entity references, duplicate GlobalIds, or combinations of multiple fault types. Build a general-purpose repair pipeline.

🎯 Tasks

Three stages, each building on the previous. All three are required.

Stage 1

Corrupt - Build Your Dataset

Source real IFC files (provided samples + public repositories), build a pipeline that introduces realistic faults, and generate your own (corrupted, clean) training pairs. Diversity of fault types and volume of data are both rewarded.

Stage 2

Model Building - Train Your Repair Model

Build an AI model (fine-tuned SLM ≤ 4 B params, or any other approach) on your training pairs. Your model must accept a corrupted IFC file and produce a repaired version. It must also output a structured fault report classifying what it found.

Stage 3

Repair - Run on Test Problems

Run your repair pipeline on the 3 provided test IFC files and submit the repaired outputs. Evaluation measures fault detection accuracy, schema validity, entity recovery, and spatial / geometric coherence against hidden ground truth.

📊 Evaluation

What we checkHow
Fault detectionDid you identify the right entities as faulty and label the fault type correctly?
Schema validityDoes your repaired file pass ifcopenshell.validate with zero errors?
Entity recoveryAre missing or broken entities correctly restored compared to ground truth?
Spatial coherenceAre building elements correctly placed and linked to their floor?

fault_report.jsonl - Schema

One JSON object per line (JSONL format), one line per test file. For each detected fault, provide the IFC entity reference (#id), a short label describing the fault category (fault_type), and a human-readable description of what is wrong. 

# fault_report.jsonl - one object per line
{
  "id": "tp_0501",
  "faults": [
    {
      "entity_id": "#802",
      "fault_type": "<your_fault_label>",
      "description": "short description of the fault"
    },
    {
      "entity_id": "#316",
      "fault_type": "<your_fault_label>",
      "description": "short description of the fault"
    }
  ]
}

🚀 Getting Started

Install IfcOpenShell - the primary toolkit you will need to read, write, validate, and manipulate IFC files: 

pip install ifcopenshell

IfcOpenShell lets you open any IFC file as a Python object, query entities by type, read and set attributes, remove or add entities, and validate the file against the IFC 2x3 schema. Study the IfcOpenShell documentation and the provided sample_hackathon_dataset_2/ to understand the data format before building your pipeline.
