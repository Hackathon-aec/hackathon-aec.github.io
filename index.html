§ 02

How to submit

5 steps · GitHub Classroom

Team lead clicks the invite link

One member per team signs in to GitHub, clicks Create a new team, and names it using the format team-<yourname> — lowercase, hyphens only. Examples: team-rocket, team-byteforce.

Lead only

Teammates join the same team

Other members click the same invite link, sign in, and select your team's name from the list. Do not create duplicate teams.

All members

Code in your team repo

Your team gets a private repo at github.com/Hackathon-aec/hackathon-submission-<team-name> pre-loaded with a README template. Push your code, commit often.

Build

Fill in the README before deadline

Replace the template with your project details. The README is your submission — it tells judges what you built, how to run it, and what they should look at first.

Document

Tag your final commit

Before 11 AM on 28 April, run git tag submission && git push --tags. This marks your locked entry. Commits after the deadline will be ignored.

Lock in

Ready to start?

One link sets up your team's private repo. Takes 60 seconds.
