🚀 How to Submit

We use GitHub Classroom to give every team a private repository. One link does everything — no manual setup, no email back-and-forth. Each team gets a starter repo pre-loaded with a README template.

Click here to set up your team repo

One person per team clicks first to create the team — others join after.

Accept Assignment →

Team lead clicks the invite link

One member per team signs in to GitHub, clicks "Create a new team", and names it using the format team-<yourname> — lowercase, hyphens only. Examples: team-rocket, team-byteforce, team-pixelpushers.

Teammates join the same team

Other team members click the same invite link, sign in, and select your team's name from the list. Do not create duplicate teams.

Code in your repo

Your team gets a private repo at github.com/Hackathon-aec/hackathon-submission-<team-name> pre-loaded with a README template. Push your code, commit often.

Fill in the README before the deadline

Replace the template README with your project details. The README is your submission — see the requirements below. Anything not in the README won't be reviewed.

Tag a final commit

Before the deadline, tag your final commit: git tag submission && git push --tags. We treat this as your locked submission. Commits after the deadline will be ignored.

📏 Rules

Team composition

  • Up to 5 members per team
  • Each person can be on only one team
  • Solo participation is allowed but not encouraged

What's allowed

  • Open-source models, libraries, and pretrained weights — credit them in your README
  • AI coding assistants (Copilot, Claude, Cursor) — disclose in your README
  • External APIs (OpenAI, Hugging Face, etc.) — note them in the README so judges can provision keys if needed

What's not allowed

  • Submitting work built before the hackathon started
  • Sharing code or training data between competing teams
  • Hard-coding answers for the provided test set

❓ FAQ

Do I have to attempt both problems?
No — pick the one you find most interesting. Teams that genuinely tackle both can submit two solutions, but a polished single submission almost always beats two half-finished ones.
What if I don't have GPU access?
Both problems can be approached with smaller models or rule-based / classical CV techniques. Free options like Google Colab, Kaggle Notebooks, and Hugging Face Spaces give you GPU time. Pretrained models (foundation models, segmentation backbones, IFC parsers) are encouraged.
Can I use ChatGPT / Claude / Copilot?
Yes — just disclose it in your README. The judging values the engineering and creativity, not whether you typed every line yourself.
What happens if my repo is empty at the deadline?
That's a no-submission. The repo must contain your code and a complete README at the deadline. We don't accept late submissions.
Will my code be made public?
Repos are private during the event. After judging, we may invite winning teams to make theirs public for a showcase — that's optional and your choice.
Who owns the IP of what I build?
You do. Code is licensed MIT by default in the starter template. You're free to take your work forward however you want after the event.
What if my whole team picks the same name as another team?
GitHub Classroom enforces uniqueness — the second team to try will get an error and can pick a different name. First-come-first-served.
Where do I ask questions during the event?
Post in the official hackathon Discord/Slack channel ([LINK]). For private questions, email [ORGANIZER_EMAIL].
